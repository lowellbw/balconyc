3D Visualization Specs

Two complementary views: analytical shadow modeling (Version A) and photorealistic hybrid (Version B)

How They Work Together

These are two views of the same underlying data — the user's building, the neighboring footprints, and the sun path. They serve different emotional purposes and appear on the same results page, likely as tabs or a toggle.

Version A: "Shadow Model"

Pure 3D extruded blocks. Every building is a geometric solid, color-coded by shadow impact. Interactive sun path animation. The analytical view — shows why the numbers are what they are. Builds trust through transparency.

Think: Google Earth meets engineering software

Version B: "Your Building"

The user's building rendered with its actual Street View texture mapped onto the 3D block. Surrounding buildings remain as simplified semi-transparent volumes. The emotional view — shows this is real, this is your home.

Think: architectural rendering meets augmented reality

Shared data pipeline. Both versions use the exact same backend query: geocode the address → fetch building footprint polygon + height → fetch neighboring footprints within 200m → compute sun positions → calculate shadow factors. The only difference is rendering.
VERSION A
A

3D Shadow Model

Purpose

Show the user exactly which buildings block their sun and when. Every building in the scene is an extruded polygon from real NYC data. The user can scrub through time and seasons to see shadows move in real-time. Buildings are color-coded by how much shadow they cast on the target balcony.

Rendering Spec

Building Geometry

// Convert each NYC footprint polygon to a Three.js ExtrudeGeometry function footprintToMesh(feature, isTarget) { const coords = feature.geometry.coordinates[0]; // exterior ring const height = feature.properties.heightroof * 0.3048; // ft → meters // Convert lat/lon to local meters (origin = target building centroid) const shape = new THREE.Shape(); coords.forEach(([lon, lat], i) => { const x = (lon - targetLon) * M_PER_DEG_LON; const z = -(lat - targetLat) * M_PER_DEG_LAT; i === 0 ? shape.moveTo(x, z) : shape.lineTo(x, z); }); const geo = new THREE.ExtrudeGeometry(shape, { depth: height, bevelEnabled: false }); // Rotate so extrusion goes up (Y-axis) instead of forward (Z) geo.rotateX(-Math.PI / 2); return geo; } // This gives you ACTUAL building shapes — not boxes. // L-shaped buildings, irregular lots, courtyards all render correctly.
ExtrudeGeometry vs BoxGeometry: The prototype used BoxGeometry (rectangular blocks) for speed. In production, switch to ExtrudeGeometry from the actual footprint polygon Shape. This means L-shaped buildings, triangular lots, buildings with courtyards — they all render with their real shape. It's a single function change and uses the data you already have.

Color Coding Logic

// Each frame, recalculate shadow influence scores and update colors Building color states: Target building → Bright blue (#3399FF) + glow outline + balcony marker High shadow impact → Red (#D03030) score > 0.5 Med shadow impact → Orange (#E07020) score 0.2–0.5 Low shadow impact → Yellow (#E8B030) score 0.05–0.2 No shadow impact → Dark gray (#556677) score < 0.05 // Colors update in real-time as user scrubs the time slider. // A building that's red at 9 AM (blocking morning sun) may be // gray at 3 PM (sun has moved past it). This is the "aha" moment.

Key insight: pre-compute the shadow summary on the backend, render interactively on the frontend. The backend runs the full 8,760-hour shadow algorithm (2-5 seconds) and returns both the raw polygon data (for 3D rendering) and the pre-computed annual/monthly shade factors (for the energy calculation). The frontend Three.js scene handles the visual shadow casting in real-time using GPU shadow maps — this is approximate but visually convincing. The actual energy numbers use the precise backend calculation.
VERSION B
B

Hybrid Street View + 3D Model

Purpose

Make it personal. The user's building is rendered with its actual Google Street View photograph texture-mapped onto the 3D geometry. Surrounding buildings appear as semi-transparent architectural volumes — present enough to show context and shadow sources, but faded so the user's building is the clear focal point. The balcony is highlighted, and optionally the AI-generated solar panel overlay appears on it.

How the Street View Texture Mapping Works

This is the core technical challenge of Version B. You're projecting a 2D photograph onto one face of a 3D extruded polygon. Here's the exact approach:

Step 1: Fetch the Street View Image

// Fetch Street View image facing the user's building facade // The balcony direction tells us which facade to photograph const balconyAzimuth = 180; // south-facing balcony // Camera should be OPPOSITE the balcony (looking at it from across the street) // So if balcony faces south, camera is south of building looking north const cameraOffset = 30; // meters from building const cameraLat = targetLat - (cameraOffset / 110540) * Math.cos(balconyAzimuth * DEG2RAD); const cameraLon = targetLon - (cameraOffset / M_PER_DEG_LON) * Math.sin(balconyAzimuth * DEG2RAD); // Heading: camera looks BACK toward the building const heading = (balconyAzimuth + 180) % 360; // opposite of balcony direction // Pitch: tilt up to capture the balcony floor level const balconyHeight = numFloors * 3.2 * userFloorFraction; const pitch = Math.atan2(balconyHeight, cameraOffset) * (180 / Math.PI); const url = `https://maps.googleapis.com/maps/api/streetview` + `?size=1024x1024` + `&location=${targetLat},${targetLon}` + `&heading=${heading}` + `&pitch=${Math.min(pitch, 45)}` // cap at 45° — higher gets distorted + `&fov=80` + `&key=${GOOGLE_API_KEY}`; // Cost: $0.007 per image

Step 2: Identify the Target Facade Edge

// From the building footprint polygon, find the edge that faces // the balcony direction. This is the edge we'll texture. function findFacadeEdge(footprintCoords, balconyAzimuth) { const targetAngle = balconyAzimuth * DEG2RAD; let bestEdge = null, bestScore = -Infinity; for (let i = 0; i < coords.length; i++) { const p1 = coords[i], p2 = coords[(i + 1) % coords.length]; // Edge direction const edgeDx = p2.x - p1.x, edgeDz = p2.z - p1.z; const edgeLen = Math.sqrt(edgeDx**2 + edgeDz**2); // Outward-facing normal (perpendicular, pointing away from building center) const normalAngle = Math.atan2(-edgeDz, edgeDx); // perpendicular // Score: how closely does this edge's normal match the balcony direction? // Also weight by edge length (prefer the main facade, not a tiny notch) const angleDiff = Math.abs(normalAngle - targetAngle) % (Math.PI * 2); const alignment = Math.cos(angleDiff); const score = alignment * edgeLen; if (score > bestScore) { bestScore = score; bestEdge = { p1, p2, edgeLen }; } } return bestEdge; // { p1, p2, edgeLen } — the edge to texture }

Step 3: Map the Image onto the Facade

// Create a PlaneGeometry sized to the facade, position it on the correct edge, // and apply the Street View image as a texture. function createTexturedFacade(edge, buildingHeight, streetViewUrl) { const facadeWidth = edge.edgeLen; // meters (from footprint) const facadeHeight = buildingHeight; // Load Street View image as texture const texture = new THREE.TextureLoader().load(streetViewUrl); texture.colorSpace = THREE.SRGBColorSpace; // Create facade plane const geo = new THREE.PlaneGeometry(facadeWidth, facadeHeight); const mat = new THREE.MeshStandardMaterial({ map: texture, roughness: 0.8, metalness: 0.05, side: THREE.FrontSide }); const mesh = new THREE.Mesh(geo, mat); // Position at edge midpoint, half-height up const midX = (edge.p1.x + edge.p2.x) / 2; const midZ = (edge.p1.z + edge.p2.z) / 2; mesh.position.set(midX, facadeHeight / 2, midZ); // Rotate to align with edge direction const angle = Math.atan2(edge.p2.x - edge.p1.x, edge.p2.z - edge.p1.z); mesh.rotation.y = angle; // Slight offset outward to prevent z-fighting with the 3D block behind const normal = new THREE.Vector3(-Math.cos(angle), 0, Math.sin(angle)); mesh.position.add(normal.multiplyScalar(0.15)); return mesh; }

Step 4: Style the Scene for Contrast

// Target building: full-color extruded geometry + textured facade targetMaterial = { color: 0xCCDDEE, // light neutral — non-textured faces roughness: 0.7, opacity: 1.0, transparent: false, // The Street View plane sits ON TOP of this face }; // Neighbor buildings: semi-transparent ghost volumes neighborMaterial = { color: 0x445566, // dark blue-gray roughness: 0.9, opacity: 0.35, // key: see-through, but shadows still cast transparent: true, depthWrite: false, // prevents transparency sorting artifacts }; // Neighbor buildings that CAST SHADOW on target: slightly more opaque shadowCasterMaterial = { color: 0x556677, opacity: 0.55, // more visible to show "this is blocking you" transparent: true, depthWrite: false, }; // Edge wireframes on all neighbors for architectural definition wireframeMaterial = { color: 0x667788, opacity: 0.25, transparent: true, };

Step 5 (Optional): Overlay AI-Generated Solar Panels

// If the AI visualization pipeline has already run (Version B, Phase 2), // the inpainted Street View image (with panels) can be used INSTEAD of // the raw Street View image. // Toggle between before/after: const [showPanels, setShowPanels] = useState(false); // Swap texture on the facade plane: facadeMesh.material.map = showPanels ? panelTexture : originalTexture; facadeMesh.material.needsUpdate = true; // This gives you a 3D "before/after" that's way more compelling than // a flat 2D image comparison — you can orbit around it.

Handling Edge Cases

Edge CaseSolution
Street View image not available for this addressFall back to Version A styling (solid colored block, no texture). Show message: "Street imagery isn't available for this location — showing modeled view."
Building is too tall — Street View can't capture upper floorsTexture the lower portion of the facade. Upper portion uses a solid color or tiled window pattern. For buildings >15 floors, consider fetching multiple Street View images at different pitch angles and stitching.
Street View image shows the wrong facadeUse the balcony direction to calculate the correct heading. If multiple panoramas exist (corners), pick the one closest to the balcony-facing side. The metadata endpoint returns available panoramas.
Irregular building shape (L-shaped, etc.)Only texture the specific edge matching the balcony direction. Other faces of the target building use solid color, same as Version A.
User on a very high floor (>20)Street View can't capture that high. Show the full building in color but annotate the balcony floor with a glowing horizontal band: "Your balcony (Floor 22)" with an arrow.
Multiple Street View dates availablePrefer the most recent image. The metadata endpoint returns date (e.g., "2023-07"). Fetch the newest.

Scene Enhancements (Nice-to-Have)

Balcony floor highlight: A glowing horizontal band at the user's floor level, visible from any camera angle. Pulses gently. On the textured facade, this appears as a semi-transparent overlay stripe at the correct height: balconyY = buildingHeight × (userFloor / totalFloors).
Shadow on facade: When a neighboring building casts shadow on the target facade, the Three.js shadow map darkens that portion of the Street View texture in real-time. The user literally sees the shadow creep across their building as they scrub the time slider. This is automatic — Three.js shadow mapping handles it with receiveShadow = true.
Ground plane: Dark with subtle street grid lines. Could optionally use a satellite tile (Google Static Maps API, $2/1K) for the block around the building, but the dark grid is cleaner and cheaper.
Depth of field: Use Three.js post-processing (BokehPass or DepthOfFieldEffect) to gently blur distant buildings. Draws the eye to the target building. Performance cost: minimal on modern GPUs.
